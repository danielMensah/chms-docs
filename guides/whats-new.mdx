**What's New** keeps you informed when Bethel ChMS ships updates. It shows release notes in the staff app so you can see new features, improvements, and fixes without leaving your workflow.

Release notes are informational only. Dismissing them does not change records, settings, permissions, or billing.

## Latest release: v0.8.0

**Tag your people** adds colour-coded tags for members and visitors.

- Create and manage custom tags from **Tags** in the sidebar.
- Assign tags to members and visitors, then see them as coloured chips across lists, details, check-in, attendance, household rosters, and the member portal.
- Filter member lists, visitor lists, reports, and demographics by tag.
- Choose **Match any** or **Match all** when report filters include multiple tags.
- Use tags in backslider rule conditions and backslider list filters.
- Export members and visitors with a **Tags** column.
- Merge duplicate members without losing tags from either record.

See [Tags](/guides/tags) for setup and usage.

## Where to find updates

Bethel ChMS shows release updates in two places:

- A **What's New** popup that can appear after you sign in or load the staff app.
- The **What's New** page at `/whats-new`, which lists release notes newest first.

Open the full release history from the staff app sidebar footer by clicking **What's New**. You can also click **See all updates** in the popup.

Treasurer users cannot open the **What's New** page. If a treasurer follows the footer link or visits `/whats-new` directly, Bethel ChMS redirects them to **Gifting & Donations**.

Unauthenticated direct visits to `/whats-new` are not redirected by the current proxy. The page loads in the app shell, but release notes are only readable to authenticated users, so unauthenticated visitors do not receive release history content.

## What the popup shows

The popup includes:

- A **What's New** title.
- A version badge, such as `v0.8.0`.
- A short release headline.
- Release items grouped as **New**, **Improved**, and **Fixed**.
- **See all updates** and **Got it** actions.

Longer release notes scroll inside the popup.

## When the popup appears

The popup appears only when Bethel ChMS has a newer published release note for your account.

Bethel ChMS does not show the popup when:

- You have already dismissed the latest release.
- The latest release was published before your account was created.
- You need to review updated terms first.
- No release notes have been published.

Only the latest release opens as a popup. If you miss several releases, Bethel ChMS shows the newest one instead of showing each missed release in sequence.

<Info>
If updated terms are pending, the terms flow takes priority. Review and accept the terms first. **What's New** can appear on a later app load if a release note still applies to your account.
</Info>

## Dismiss a release note

Any dismissal marks the release as seen for your account.

You can dismiss the popup by:

- Clicking **Got it**.
- Clicking **See all updates**.
- Closing the popup.
- Pressing **Escape**.
- Clicking outside the popup.

After you dismiss a release, Bethel ChMS saves that dismissal to your account. The same release does not reappear when you reload, switch browser, or use another device.

## Read all release notes

The **What's New** page lists every published release note in reverse chronological order.

Each release shows:

- The version.
- The release headline.
- The publication date.
- The **New**, **Improved**, and **Fixed** groups that have items for that release.

If no releases have been published, the page shows **No releases yet. Check back soon.**

## If you do not see What's New

You may not see a popup every time Bethel ChMS changes. This usually means one of these is true:

- You already dismissed the latest release.
- Your account was created after the release was published.
- You need to review updated terms first.
- The release does not include a published release note yet.

Use the **What's New** link in the sidebar footer to review release notes manually.
