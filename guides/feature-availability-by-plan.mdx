Bethel ChMS uses one Standard plan. Every product feature is included for every organisation.

## Included features

The Standard plan includes:

| Feature | Availability |
| --- | --- |
| Gifting and donations | Included |
| CSV member import | Included |
| Inline custom field creation | Included |
| Member and visitor tags | Included |
| Organisation dashboard | Included |
| Branch performance comparison | Included |
| Pastoral activity recording | Included |
| Pastoral activity reporting | Included |
| Backsliders | Included |
| Member QR code display | Included |
| Occurrence QR scanning | Included |
| Admin users | Included with no enforced cap |
| Automations | Included with no enforced cap |

The app may still hide a feature from roles that do not have permission to use it. For example, branch admins and editors do not see **Gifting & Donations**.

## Branch limit

Branches are the only enforced usage limit in the Standard plan.

New organisations start with a branch limit of 1. The **Plan & Usage** card in **Organisation Settings** shows your current branch usage and branch limit.

When you reach the branch limit and try to create another branch, Bethel ChMS shows **Branch limit reached** with **Contact us**. No branch is created until the limit is increased.

<Note>
Billing is also branch-based. See [Billing and subscription](/guides/billing-and-subscription) for pricing and branch billing behavior.
</Note>

## Member portal access

Member portal access is included in the Standard plan, but it is not available yet.

If you open a portal-gated flow before rollout, Bethel ChMS shows **Member portal access is coming soon** with **Contact us**.

## Feature unavailable page

Bethel ChMS keeps a shared feature availability page for gated routes.

Most features are included and should not send you there. If a feature is temporarily unavailable, the page explains that the feature is included in your Standard plan and asks you to contact support.

## What is not self-serve

Some changes are handled by Bethel support instead of self-serve controls.

Contact support when you need to:

- Increase your branch limit.
- Resolve a feature availability message.
- Ask about member portal rollout.
- Update billing details for a manually managed account.

Support mail links use `hello@bethelware.app`.
